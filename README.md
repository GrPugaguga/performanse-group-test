Проект разделен на модули отвечающие за авторизацию, юзеров и статьи 

Дополнительная логика для соответсвия 'авторизованные пользователи могут создавать, изменять, удалять статьи и пользователей'

  - не авторизованный пользователь видит только публичные статьи 

  - авторизованный пользователь видит все статьи 
  
  - при регистрации нового пользователя он получает роль user 

  - любой пользователь с ролью admin может повысить любого пользщователя с ролью user до admin

  - user может редактировать только свои статьи, admin может редактировать любые статьи

  - user может удалить только себя, admin может любого пользователя с ролью user , либо себя 

API endpoints 

  auth/

    post login - логин с получением jwt токена доступа 

    post register - регистрация с получением jwt токена доступа 

    get profile - получение данных о себе


  users/ 
    get users - получение списка всех пользователей 

    get users/{id} - получение данных о конкретном пользователе 

    delete users/{id} - удаление пользователя 

    post users/{id}/promote - повышение роли пользователя другого до админа  


  articles/ 
    post articles - создание статьи

    get articles - получение всех статей, авторизованные пользователи видят все, не авторизованные только публичные. query строка принимает список тегов для фильтрации через запятую 

    get articles/{id} - получение отдельной статьи

    patch articles/{id} - обновление статьи если ты владелец либо админ

    delete articles/{id} - удаление статьи если ты владелец либо админ
